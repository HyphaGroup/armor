# Civil Society Security Guide Summaries

Analysis of major civil society digital security guides for potential refinements to the threat modeling methodology.

---

## 1. Security in a Box (Front Line Defenders / Tactical Tech)
**URL:** https://securityinabox.org/en/
**Focus:** Digital security tools and tactics

### Structure
- **Assess & Plan**: Physical security, multiple identities, private communication, travel security
- **Passwords**: Strong passwords, password managers, 2FA
- **Communication**: Social media, secure email, messaging apps, account compromise
- **Phones & Computers**: Malware, Android/iOS/Windows/Mac/Linux protection
- **Internet Connection**: How internet works, safer browsing, circumvention, anonymity
- **Files**: Secure storage, backup, destroying sensitive information

### Key Approach
- Very practical, tool-focused
- Organized by "what you need to protect" (action-oriented)
- Strong emphasis on specific tools and how-tos
- Available in 13+ languages

### Notable Elements
- Clear separation between concepts and tool guides
- Device-specific guidance (Android vs iOS vs desktop platforms)
- Strong coverage of anonymity and circumvention

## 2. EFF Surveillance Self-Defense (Electronic Frontier Foundation)
**URL:** https://ssd.eff.org/
**Focus:** Protecting against surveillance

### Structure
- **Basics**: Overview of surveillance, starter resources
- **Tool Guides**: Instructions for installing secure applications
- **Security Scenarios**: Situation-specific advice and plans

### Security Planning Framework (Key Module)
Six questions for security planning:
1. What do I want to protect? (assets)
2. Who do I want to protect it from? (adversaries)
3. How bad are the consequences if I fail?
4. How likely is it that I will need to protect it?
5. How much trouble am I willing to go through?
6. **Who are my allies?** (unique addition)

### Key Approach
- Individual-focused (personal security)
- Threat modeling as core concept
- Scenario-based guides (activist, journalist, LGBTQ youth, etc.)
- Emphasis on encryption and communications

### Notable Elements
- "Security is a process" framing
- Regular review cadence recommendation
- The "allies" question (community aspect)
- Graduated approach for different expertise levels

## 3. Holistic Security (Tactical Tech)
**URL:** https://holistic-security.tacticaltech.org/
**Focus:** Integrated well-being, digital, and organizational security

### Structure (Cyclical Process)
1. **Prepare**: Recognize existing practices, individual responses to threat, beliefs/values, team dynamics
2. **Explore**: Context analysis, situation monitoring, actors, cataloguing information, security indicators, threat identification
3. **Strategise**: Responding to threats, new security approaches, security plans/agreements, organizational security
4. **Act**: Practical skills for specific activities (protests, etc.)

### Key Approach
- "Well-being in action" as core framing
- Integration of psycho-social, physical, and digital security
- Gender-aware approach
- Organizational/team focus (not just individual)
- Explicit connection between security and sustainability of activism

### Notable Elements
- **Well-being as political resistance**: "Caring for myself is not an indulgence"
- Security indicators (early warning signs)
- Do-No-Harm approach integration
- Stress and trauma acknowledgment
- Team/organizational security culture
- Available as downloadable PDF with trainers' manual

## 4. Digital First Aid Kit (CiviCERT / RaReNet)
**URL:** https://digitalfirstaid.org/
**Focus:** Emergency response to digital incidents

### Structure
Problem-based navigation:
- Lost or stolen devices
- Account access issues
- Suspicious device behavior
- Online harassment
- Surveillance concerns
- And more specific emergency scenarios

### Key Approach
- Triage/diagnostic model
- Connects users to support organizations
- Emergency-focused (reactive, not just preventive)
- Self-care integration for dealing with incidents

### Notable Elements
- **Decision tree navigation**: Walk users through diagnosis
- Connect to human help (CiviCERT teams)
- Available in 14 languages
- Self-care tips for emotional impact
- Open-source, community-maintained

## 5. Front Line Defenders Workbook on Security
**URL:** https://www.frontlinedefenders.org/en/workbook-security
**Focus:** Risk assessment and protection planning for HRDs

### Structure
1. Introduction to HRD security
2. Assessing Risk
3. Analyzing Threats
4. Well-being and Stress
5. Creating Security Plans
6. Understanding Your Context

### Key Approach
- Workshop-based methodology
- Practical exercises and checklists
- HRD-specific (human rights defenders focus)
- Integrated well-being

### Notable Elements
- Appendices with practical checklists (office security, home security)
- SWOT analysis for security planning
- Context analysis questions
- Protection measures for clients/witnesses/sources
- 50+ countries of practitioner input

## 6. CPJ Journalist Security Guide / Digital Safety Kit
**URL:** https://cpj.org/2019/07/digital-safety-kit-journalists/
**Focus:** Journalist safety (physical and digital)

### Structure
- Basic Preparedness
- Assessing and Responding to Risk
- Technology Security
- Armed Conflict and Organized Crime
- Stress Management

### Digital Safety Kit Modules
1. Protect your accounts
2. Phishing defense
3. Device security
4. Encrypted communications
5. Safe internet use
6. Crossing borders

### Key Approach
- Journalist-specific threats (legal, physical, digital)
- Role-specific guidance (freelancers, editors, etc.)
- Assignment-based risk assessment
- Pre-assignment preparation emphasis

### Notable Elements
- Border crossing guidance
- Stress and trauma for covering traumatic events
- Freelancer-specific considerations
- Newsroom security culture

## 7. SAFETAG (Internews)
**URL:** https://safetag.org/
**Focus:** Organizational security audits for civil society

### Structure
- Preparation
- Context Research
- Capacity Assessment
- Reconnaissance
- Policy Review
- Network Mapping
- Assessment & Reporting

### Key Approach
- Professional audit framework
- Capacity-aware (designed for under-resourced orgs)
- Collaborative risk modeling with organizations
- Focus on what matters to THEM (not generic)

### Notable Elements
- Practitioner curriculum
- Audit methodology (not self-assessment)
- Capacity-aware recommendations
- Open-source methodology
- Builds internal capacity alongside identifying issues

## 8. General Best Digital Security Practices (Organizational Policy Template)
**Source:** Practitioner-developed organizational security policy
**Focus:** Actionable security requirements for organizations

### Structure
- **Immediate Actions**: Delete old accounts, 2FA, no PII exposure
- **Tools & Services**: Specific platform recommendations (encouraged/discouraged)
- **Email Security**: Requirements and rationale
- **Google Workspace**: Detailed configuration requirements
- **Signal**: Configuration and usage requirements
- **Zoom**: Security settings requirements
- **Hardware & Devices**: Desktop, laptop, mobile recommendations
- **Social Media**: Privacy recommendations
- **Personal Safety**: Individual protection measures
- **Getting Help**: Support resources
- **Checklist**: Self-assessment tool

### Key Approach
- Very prescriptive (specific tools, settings, requirements)
- Operational policy format (requirements vs. recommendations)
- Platform-specific guidance (what to use, what to avoid)
- Includes rationale for each recommendation
- Ends with actionable checklist

### Notable Elements
- **Encouraged/Discouraged technology lists**: Clear guidance (use Signal, not Slack/SMS)
- **Specific configuration requirements**: Exact settings for Zoom, Signal, Google Workspace
- **Personal safety section**: Unique coverage of doxxing prevention, credit freezes, family code words
- **Self-assessment checklist**: Practical verification tool
- **Security rationale**: Explains "why" for each recommendation
- **Data broker removal services**: Addresses proactive privacy protection

### Unique Contributions
- Most operational/prescriptive of all guides reviewed
- Specific tool configuration guidance
- Personal safety beyond digital (credit freezes, family verification)
- "Discouraged technologies" framing (clearer than just "use X")
